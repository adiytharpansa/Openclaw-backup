## 🔧 How to Use

### Basic Usage

**1. Simple Request:**
```
"Bantu build REST API untuk todo app"
```
→ Aku akan autonomously:
- Design architecture
- Choose tech stack
- Write code
- Test
- Document
- Deliver complete solution

**2. With Context:**
```
"Bantu fix bug memory leak di image processor"
Stack: Node.js, Express
Database: MongoDB
```
→ Aku akan:
- Analyze existing code
- Identify root cause
- Implement fix
- Add regression tests
- Document the solution

**3. Autonomous Mode:**
```
/autonomous "Create a CLI backup tool for PostgreSQL"
```
→ Full ownership, dari planning sampai delivery

### Autonomy Levels

**Level 1 - Guided (Complex tasks):**
- Plan first, get approval
- Update at checkpoints
- Ask before major decisions

**Level 2 - Semi-Auto (Standard tasks):**
- Plan & execute independently
- Update on milestones only
- Escalate only blockers

**Level 3 - Full Autonomy (Routine tasks):**
- Complete ownership
- No updates unless asked
- Deliver when 100% ready

### Commands

**Start Autonomous Work:**
```
/autonomous <task description>
```
Example:
```
/autonomous "Add user authentication to my React app"
```

**Check Status:**
```
/status
```
Shows progress, current phase, ETA, blockers

**Pause Work:**
```
/pause
```
Saves state, stops temporarily

**Resume Work:**
```
/resume
```
Continues from last checkpoint

**Abort Work:**
```
/abort
```
Stops immediately, cleans up
