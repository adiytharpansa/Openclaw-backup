# Reasoning Enhancer

Buat jawaban yang **pikirannya dalam & logis**, bukan asal jawab.

## 🎯 Core Function

Skill ini nge-force AI buat:
1. **Think deeper** - gak asal jawab
2. **Check logic** - validasi reasoning sebelum output
3. **Multiple perspectives** - liat dari 3+ angle
4. **First principles** - break down ke fundamental truths
5. **Bias detection** - identify & avoid cognitive biases

## 🧠 Framework

### Step 1: Problem Decomposition
Break problem into parts:
- What's the core question?
- What assumptions am I making?
- What information is missing?
- What are the constraints?

### Step 2: Multi-Perspective Analysis
Always consider:
- **Perspective A** - Standard/obvious view
- **Perspective B** - Contrarian/opposite view  
- **Perspective C** - Third option/synthesis

Example for "Should I deploy now?":
- A: Yes, we have testing done
- B: No, wait for more testing
- C: Partial deployment to 10% users

### Step 3: First Principles Thinking
Strip to fundamentals:
- What are we absolutely sure of?
- What's derived from assumptions?
- Can I prove this step-by-step?

### Step 4: Logic Chain Verification
Check each inference:
- Does A → B really follow?
- Is there a missing link?
- Could B happen without A?

### Step 5: Bias Detection
Watch for:
- Confirmation bias (seeking supportive evidence)
- Sunk cost fallacy (continuing due to prior investment)
- Overconfidence (overestimating knowledge)
- Availability heuristic (relying on immediate examples)

## 📝 Output Format

For complex questions, use this structure:

```
## Analysis

**Core Question:** [Restate the question]

**Key Assumptions:**
1. [Assumption 1]
2. [Assumption 2]

## Perspectives

### Viewpoint A
[Analysis from perspective A]

### Viewpoint B
[Analysis from perspective B]

### Viewpoint C
[Analysis from perspective C]

## Logical Chain
- Premise 1: [Evidence]
- → Intermediate conclusion: [Deduction]
- → Premise 2: [Evidence]
- → Final conclusion: [Answer]

## Biases Checked
✓ Not confirmation bias (considered opposing views)
✓ Not sunk cost (evaluating current situation)
✓ Confidence level: [Low/Medium/High]

## Recommendation
[Final answer with confidence level]
```

## 🔧 When to Use

### Use Reasoning Enhancer When:
- ✅ Complex decisions (investment, hiring, strategy)
- ✅ Technical architecture choices
- ✅ Conflict resolution
- ✅ Problem-solving with multiple variables
- ✅ When you say "I'm not sure" or "this is tricky"
- ✅ Before giving important advice

### Don't Overuse When:
- ❌ Simple factual questions
- ❌ Quick confirmations
- ❌ When user wants speed over depth

## 🎨 Depth Levels

### Light Analysis (for routine questions)
- Quick multi-perspective check
- 2-3 key assumptions noted
- Confidence level stated

### Medium Analysis (for standard decisions)
- Full 3-perspective framework
- Basic logic chain
- Bias check

### Deep Analysis (for major decisions)
- Extensive perspective breakdown
- First-principles derivation
- Complete logic validation
- Risk assessment
- Alternative scenarios

---

**Rule of thumb:** If it takes >30 seconds to think, use Reasoning Enhancer.
